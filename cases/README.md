# PDF Case Set

这组样本用于复现和比对双层 PDF 问题，重点覆盖：

- 旋转元数据扫描型 PDF
- 历史生成的“较好版 / 失败版”双层 PDF
- 已有文本层的电子 PDF 参照样本

## 文件说明

### `case01_raw_acceptance_rotate270.pdf`

- 原始验收文档样本
- 1 页
- `Rotate = 270`
- 横向底图 + 旋转元数据显示为竖版
- 无原生文本层

用途：
- 验证单页旋转扫描件在当前叠字逻辑下是否仍有高亮错位

### `case02_raw_acceptance_rotate90_multi.pdf`

- 原始多页验收文档样本
- 3 页
- 每页 `Rotate = 90`
- 横向底图 + 旋转元数据显示为竖版
- 无原生文本层

用途：
- 验证多页旋转扫描件
- 验证是否存在页级文字层缺失

### `case03_generated_perfect_rotate0.pdf`

- 桌面历史生成产物“完美版”
- 3 页
- `Rotate = 0`
- 三页均能提取文字层

用途：
- 作为当前最接近正确结果的对照样本
- 服务器环境回查时，优先对照这份产物的生成思路

### `case04_generated_alignment_failed.pdf`

- 桌面历史生成产物“对齐版”
- 3 页
- `Rotate = 90`
- 提取字符分布：`1006 / 0 / 0`

用途：
- 作为失败样本对照
- 说明问题不只是“高亮偏移”，还存在页级文字层缺失

### `case05_einvoice_text_layer_reference.pdf`

- 电子发票参照样本
- 1 页
- `Rotate = 0`
- 原生文本层可直接提取

用途：
- 作为“已有文本层 PDF”的参照
- 不适合作为扫描件双层 PDF 对齐问题的主测试样本

## 使用建议

1. 优先用 `case01` 和 `case02` 复现问题。
2. 用 `case03` 和 `case04` 对照历史生成效果。
3. `case05` 只用于区分“电子 PDF”和“扫描型 PDF”的处理路径。
